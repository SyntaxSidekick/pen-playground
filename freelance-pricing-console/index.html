How this pricing console works

What this tool does

This console helps you move from “hourly guessing” to a repeatable pricing model. You define services and overheads, then layer overhead %, project buffer %, and profit % on top.

1 — Add services

  1. Create one line per project phase (Discovery, UX, Front-end build, QA).
  2. Set an hourly rate that reflects the value of that phase.
  3. Estimate hours realistically, including reviews and communication.

2 — Add overhead expenses

  1. Use overhead items for tools, subscriptions, admin, tax, and ops.
  2. Think in “per project” slices (for example, a portion of Figma, hosting, etc.).
  3. The raw overhead total is for your eyes only; clients don’t see this number.

3 — Apply overhead %, buffer %, and profit %

  • Overhead % applies a percentage of your overhead total to the price.
  • Project buffer % is a contingency layer on top of services + overhead.
  • Desired profit % is applied to the buffered base cost.

4 — Read the summary & print

  • Base cost = services + overhead charge + buffer.
  • Recommended project price = base cost + profit.
  • Use Print quote to generate a client-facing summary PDF.
